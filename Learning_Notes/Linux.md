## Basic Command
| 명령어 | 의미 | 사용 방법 |
| --- | --- | --- |
| `pwd` | 현재 위치 확인 | `pwd` |

| `tail` | 파일 뒷 부분 보기 | `tail 파일명` |

| `mv` | 파일 이동 or 파일명 변경 | `mv 변경대상파일 변경할이름` |

## echo
echo 터미널에 글자를 출력하는 명령어
- echo `>`: 기존 내용을 덮어쓰고 저장

| 명령어 | 의미 |
| --- | --- |
| `echo "출력 내용" > 파일` | `"출력 내용"`을 `파일`에 덮어쓰고 저장 |
| `echo "출력 내용" >> 파일` | `"출력 내용"`을 `파일` 뒤에 내용 추가 |

| --- | --- | --- |

# Redirection
- 터미 명령어 실행 결과를 파일에 저장할 떄 사용
```bash
echo "server start" >> server.log
# server.log "Server Start" 내용 저장
```
## Pipe, Grep
파이프는 `|`로 사용, 앞 명령어의 결과를 뒤로 넘기는 기능
```bash
cat server.log | grep "error"
# server.log 내용을 read해서 error가 들어간 줄만 골라내는 명령어
```

## 항목별 사용 정리
| 항목 | 의미 | 예시 |
| --- | --- | --- |
| > | 출력 결과를 파일에 저장(덮어쓰기) | `명령어` `출력 결과` > `저장 대상 파일` |
| >> | 출력 결과를 파일 끝에 추가 | `명령어` `출력 결과` >> `저장 대상 파일` |
| stdout | 표준 출력(명령어 출력 결과) | 명령어 |
| log | 서버 or 프로그램 실행 기록 | *.log |

# 사용 권한
누가(owner・group・others) 무엇을(read・write・execute)할 수 있는지 정하는 규칙

## 권한
| 권한 | 의미 |
| --- | --- |
| `r` | `Read` 읽기 |
| `w` | `Write` 쓰기 |
| `x` | `eXecute` 실행 |

## 권한 대상
| 대상 | 의미 |
| owner | 파일 소유자 |
| group | 파일 소유자가 속한 그룹 |
| others | 그 외 사용자 |

## chmod
- 파일이나 디렉토리 권한을 변경하는 명령어
``` bash
# 실행(현재 디렉토리 파일들의 권한 확인)
ls -l
# 결과
-rw-r--r-- 1 song
```

## 권한 부여 및 확인
- 권한은 아래와 같이 분리할 수 있다.

```bash
-   rwx   r-x   r--
①   ②     ③     ④
```
| 위치 | 의미 |
| --- | --- |
| 1번째 | 파일 종류(-: 일반파일 / d: 디렉터 리 / l: 심볼릭 링크) |
| 2~4번째 |	소유자(owner) 권한 |
| 5~7번째 |	그룹(group) 권한 |
| 8~10번째 |	기타 사용자(others) 권한 |

- 권한 부여는 아래와 같이 할 수 있다.
### 1) chmod `+r, w, x` 파일명
- chmod를 통해 read, write, execute 추

### 2) chmod `숫자` `숫자` `숫자`
- 숫자 4~7로 권한 부여 가능
- 주로 스크립트는 `755`, 일반 파일은 `644`로 부여함

| 숫자 | 의미 |
| --- | --- |
| 7 | r + w + x |
| 6 | r + w |
| 5 | r + x |
| 4 | r |
